# 顶部标题栏拖拽区域修复设计

## 问题

Aurora 主题中心把 `#heige-codex-skin-menu` 变成覆盖整个窗口的固定定位容器，同时在该容器上设置了 `-webkit-app-region: no-drag`。即使容器使用 `pointer-events: none`，Electron 仍会在窗口拖拽区域计算中把这层视为不可拖拽区域，导致换肤后点击 Codex 顶部空白横条无法拖动窗口。

旧版菜单只在顶部入口按钮和面板等真实交互区域设置 `no-drag`，因此标题栏其他区域可以沿用 Codex 原生拖拽行为。

## 修复

从全屏根容器 `#heige-codex-skin-menu` 移除 `-webkit-app-region: no-drag`。

继续在以下真实交互元素上保留 `no-drag`：

- 顶部主题入口按钮；
- 主题中心弹窗及遮罩；
- 弹窗内按钮、开关和其他可操作控件。

不新增人工 `drag` 区域，不猜测 Codex 标题栏高度，也不改变主题中心的位置、尺寸、层级或点击行为。

## 预期行为

1. 主题中心关闭时，除主题入口按钮本身外，Codex 顶部原生横条可以拖动窗口。
2. 点击主题入口按钮仍能打开主题中心，不会被窗口拖拽吞掉。
3. 主题中心打开后，遮罩和弹窗保持可点击、不可用于拖动窗口。
4. 隐藏后的迷你入口仍可点击恢复。
5. 原生界面和各套皮肤的主题切换行为不受影响。

## 测试

增加样式契约测试：

- 全屏根容器不得声明 `-webkit-app-region: no-drag`；
- 顶部主题入口必须声明 `-webkit-app-region: no-drag`；
- 主题中心遮罩或弹窗必须声明 `-webkit-app-region: no-drag`，避免打开面板时交互被窗口拖拽吞掉。

运行完整 Node 测试套件，并在支持 Electron 真机验证时检查顶部横条拖动、入口点击和主题中心交互。
