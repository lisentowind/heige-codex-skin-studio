# 阅读增强可靠性与性能优化设计

## 问题与证据

当前阅读增强只把底色加在 `data-local-conversation-final-assistant` 上。真实 Codex renderer 取证显示，同一对话中既有最终回复，也有过程回复：

1. 最终回复拥有 `data-local-conversation-final-assistant`。
2. 过程回复没有这个属性，因此保持透明。
3. 两类回复都拥有 `data-response-annotation-conversation`。
4. 每个最终回复内部恰好有一个 `data-response-annotation-conversation`。

这解释了同一对话中蒙版间歇生效的现象。开关、根节点属性和本地存储均正常，故障位于 CSS 覆盖范围。

现有蒙版还直接画在外层容器上，正文几乎贴着左边缘开始。外层与内部工具卡片的几何边界不同，使左上角显得锐利。26px 大阴影会扩大滚动时的绘制区域，对阅读帮助有限。

## 方案比较

### 方案一：为旧选择器补充过程回复选择器

同时给最终回复外层和过程回复正文加底色。改动最小，但最终回复会出现嵌套的双层蒙版，需要继续编写排除规则，视觉和维护成本都更高。

### 方案二：运行时观察并标记回复容器

使用 MutationObserver 追踪新消息并给外层容器加属性。可以适应更多 DOM 变化，但会新增常驻观察器、节点遍历和清理逻辑，不符合性能优先和产品轻量约束。

### 方案三：统一使用回复正文语义节点

让最终回复外层继续透明，只在 `data-response-annotation-conversation` 上绘制蒙版。该节点同时覆盖最终回复与过程回复，不需要脚本、观察器或额外 DOM。

采用方案三。

## 样式设计

关闭状态下，最终回复外层和回复正文节点都保持透明，避免 Codex 原生样式或旧版本样式残留。

开启状态仅匹配：

```css
:root[data-heige-readability="on"] [data-response-annotation-conversation]
```

具体规则：

1. 使用现有 `--heige-surface` 的 90％主题自适应底色。
2. 使用现有 `--heige-text` 保证文字颜色与主题一致。
3. 使用 1px 低对比主题边框和 22px 圆角。
4. 使用 `padding: 14px 16px 12px`，让正文和工具内容与四角保持稳定留白。
5. 使用 `box-sizing: border-box`，不扩大回复区域总宽度。
6. 不使用阴影、`backdrop-filter` 或伪元素。
7. 不使用 `overflow: hidden`，避免裁剪代码操作、菜单、工具卡片或焦点轮廓。

## 状态与数据流

开关逻辑不变。用户切换阅读增强时，仍只发生三件事：

1. 更新根节点 `data-heige-readability`。
2. 写入一个 `localStorage` 值。
3. 通过现有 BroadcastChannel 同步同源 renderer。

本次不增加后台 API、控制器请求、网络请求或主题格式字段。

## 性能边界

1. 不增加 MutationObserver。
2. 不增加滚动、输入、resize 或动画监听器。
3. 不增加运行时节点查询或节点创建。
4. 不使用大面积高斯模糊。
5. 移除回复蒙版的大阴影，减少长卡片的额外绘制范围。
6. 样式只在浏览器现有节点上进行静态选择器匹配。

## 回归测试

自动化测试必须证明：

1. 开启规则只以 `data-response-annotation-conversation` 为蒙版目标。
2. 最终回复外层不再绘制开启态蒙版。
3. 关闭状态同时清理最终回复外层和正文节点。
4. 蒙版有 90％底色、22px 圆角、明确内边距和 `border-box`。
5. 蒙版没有阴影、模糊、动画、伪元素或运行时观察器。
6. 既有开关、持久化、多 renderer 同步和清理测试继续通过。

## 真实验收

安装新构建包后，在真实 Codex 长对话中验证：

1. 最终回复和过程回复都具有非透明主题底色。
2. 同一回复只出现一层蒙版。
3. 左右上角圆角和正文留白对称。
4. 关闭后所有 AI 正文恢复透明，重新开启后全部恢复。
5. 重启后阅读增强与皮肤常驻状态正确恢复。
6. 回复计算样式中的 `backdrop-filter` 为 `none`，`box-shadow` 为 `none`。
7. renderer 不新增观察器、滚动监听器或定时刷新任务。
8. 长对话滚动和输入无明显卡顿，控制器日志无新增错误。
