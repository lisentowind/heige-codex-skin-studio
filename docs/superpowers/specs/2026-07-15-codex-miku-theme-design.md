# Codex Miku 主题设计

## 目标

把本机 Codex Desktop 的官方界面改造成参考图所示的冰蓝、青绿、粉紫主题，同时保留现有任务、登录态、插件、终端、浏览器和自动更新能力。

## 视觉方向

主题采用「未来糖霜玻璃」风格。浅冰蓝作为大面积底色，青绿色负责主要操作和选中态，粉紫色只用于高光、徽标和局部渐变。主内容面板使用半透明白色，描边带轻微蓝紫色泽，圆角整体放大，但不改变现有信息架构。

界面不嵌入参考图中的初音未来人物素材，避免版权、可读性和工作场景干扰。参考图只作为色彩、质感、层次和装饰语言来源。

## 技术方案

主题以 CSS 变量和少量稳定选择器覆盖实现。安装器定位官方 `/Applications/ChatGPT.app/Contents/Resources/app.asar`，解析 `webview/index.html`，把原启动样式区替换成等长主题样式。等长替换不会重排 ASAR 内其他资源偏移，也不修改业务 JavaScript。

修改前创建带版本和 SHA-256 信息的备份。重复安装具有幂等性。恢复脚本只在备份哈希匹配时还原。官方升级导致 ASAR 结构变化时，安装器必须拒绝操作并提示重新适配。

## 覆盖范围

1. 应用底色、侧栏、主内容面板和右侧面板。
2. 输入框、按钮、选中态、悬浮态、徽标和滚动条。
3. 代码块、终端、Diff 和弹窗的颜色兼容。
4. 浅色和深色设置都强制使用同一套浅色主题，避免切换系统外观后变黑。
5. 启动加载画面使用同一套配色。

## 验收标准

1. 主题 CSS 包含完整色彩令牌，且小于可替换样式区容量。
2. 安装前后 ASAR 文件大小完全一致。
3. 除 `webview/index.html` 的目标样式区外，其余资源字节不变。
4. 安装器、恢复器和主题结构测试全部通过。
5. `codesign --verify` 的结果被明确记录。若官方签名因资源修改失效，必须在交付说明中如实提示，不宣称签名仍有效。

## 风险与回滚

macOS 的官方资源签名会覆盖 `app.asar`。资源修改可能使 `codesign --verify` 报错，且官方更新会覆盖主题。原始 ASAR 会被保存在用户目录的独立备份中，恢复时不需要重新安装 Codex。主题项目保留一条重新应用命令，供官方更新后使用。
