# HeiGe Codex 主题中心升级设计

## 目标

把现有顶部 30px 圆形画笔按钮和 330px 纵向菜单升级为大型「HeiGe 主题中心」。新界面要让用户通过真实图片缩略图快速辨认主题，支持连续试用，并保持刚完成的即时视觉切换、后台耐久保存、失败回滚和常驻控制安全边界。

最终视觉采用已确认的 A 版「极光画廊」。不采用后续「珍珠影廊」实验版。

## 设计原则

1. 主题图片是第一视觉信息，名称和配色是第二信息。
2. 面板属于 Codex 界面的一部分，不做成独立后台或内容管理系统。
3. 珍珠白玻璃为主体，青色表示交互与成功，樱粉只用于小面积选中强调。
4. 不加载网络资源、外部字体、框架或运行时依赖。
5. 不增加持续动画。状态变化通过颜色、描边、文字和图标表达。
6. 视觉即时切换与 authoritative state 提交继续分离，未确认状态不得冒充已保存。

## 入口按钮

入口继续固定在 Codex 顶部中央，避开 macOS 和 Windows 窗口控制区。

按钮从圆形画笔 emoji 升级为高度 30px 的玻璃胶囊：

1. 左侧为 19px 当前主题圆形缩略图；原生模式使用中性 Codex 图形。
2. 中间显示「主题」。
3. 右侧使用小型星形状态符号。
4. 背景为高通透浅色玻璃，边界使用当前主题 accent 的低透明混色。
5. 隐藏入口后保留现有 24px 可点击命中区域，不制造不可恢复状态。

按钮提供 `aria-label`、`aria-expanded` 和 `aria-controls`，关闭主题中心后恢复焦点。

## 大型主题中心

主题中心使用固定遮罩和居中对话框，桌面宽度约为视口的 70%，上限 1100px；高度不超过视口减 72px。背景保持可见但被轻度压暗和模糊。

对话框分为三个固定区域：

1. 头部：品牌标记、标题「HeiGe 主题中心」、说明、保存状态和关闭按钮。
2. 中部滚动区：当前主题横幅、快速操作、自定义主题和内置主题网格。
3. 底部：皮肤常驻状态、常驻开关和隐藏入口操作。

头部和底部不随主题列表滚动。

### 当前主题横幅

横幅使用当前主题 hero 图片作为大预览，左侧增加可读性渐变，显示：

1. 「正在使用」状态。
2. 主题名称。
3. 主题 ID 和由 manifest 颜色生成的配色圆点，不新增难以维护的营销文案字段。
4. 当前主题标记。

切换主题时横幅立即更新。后台提交期间显示「正在保存」，成功后显示「已保存」；失败回滚后显示安全错误提示。

### 快速操作

当前主题横幅下方放置两个并列操作：

1. 「恢复原生 Codex」：使用中性预览图，进入现有原生主题提交流程。
2. 「用图片创造主题」：打开现有本机文件选择器和压缩取色流程。

两个操作都是原生 `button`，不得使用可点击 `div`。

### 自定义主题

仅在存在有效自定义主题时显示「我的主题」区域。卡片使用本机压缩图片作为预览，并提供独立删除按钮。

删除操作不得因事件冒泡触发主题切换。删除当前自定义主题时沿用现有恢复原生逻辑，并清理本机兼容存储。

### 内置主题网格

内置主题在桌面宽度下使用三列卡片，中等宽度两列，窄窗口一列。

每张卡片包含：

1. 真实 hero 缩略图。
2. 主题名称。
3. 规范化后的主题 ID 和最多三个 manifest 配色圆点。
4. 当前主题的选中描边、勾选标记和 `aria-pressed="true"`。

单击卡片立即应用当前 renderer 的目标 CSS，主题中心保持打开。主题卡片在同一请求未完成时进入不可重复提交状态，但仍清楚显示当前视觉预览。

## 预览图数据

主题 hero 已作为 data URL 包含在每个主题的生成 CSS 中。新界面不再次把完整图片嵌入 payload，避免 CDP 注入脚本体积翻倍。

构建菜单数据时保留现有主题 CSS。renderer 端使用严格、只读的解析函数，从受信任的生成 CSS 中提取第一个符合本项目图片 data URL 规则的 hero URL作为预览。解析失败时显示由主题 accent、secondary 和名称首字符生成的降级预览，不阻断主题切换。

自定义主题直接使用已经通过浏览器上传校验并压缩的 `dataUrl`。原生 Codex 使用不依赖图片资源的中性渐变预览。

## 交互与状态流

### 打开与关闭

1. 点击入口打开主题中心。
2. 打开时把焦点移动到关闭按钮或当前主题卡片。
3. `Esc`、关闭按钮或点击遮罩关闭。
4. `Tab` 和 `Shift+Tab` 在对话框可聚焦元素内循环。
5. 关闭后焦点返回入口。

### 主题切换

1. 记录点击前的主题 ID。
2. 当前 renderer 立即应用目标主题，仅更新视觉，不提前写兼容存储或广播。
3. 卡片、横幅和保存状态同步更新为目标主题和「正在保存」。
4. 控制端点在耐久锁和 revision 校验内提交 state。
5. ACK 成功后写兼容选择记录、广播跨窗口主题并显示「已保存」。
6. HTTP 拒绝、revision 冲突、控制器兜底超时或更新被更高 revision 取代时，恢复点击前主题并显示错误。

同一时刻只允许一个正式主题提交。现有 renderer fallback 请求协议、控制 capability 和超时边界保持不变。

### 常驻与隐藏入口

常驻开关保留现有 authoritative controller 协议、关闭确认和失败补偿语义，只改变布局和文案层级。

隐藏入口继续写入现有本机隐藏键。隐藏后主题中心立即关闭，并保留最小可恢复点击目标。

## 响应式布局

1. 宽度大于等于 980px：三列主题卡片，快速操作两列。
2. 宽度 680px 至 979px：两列主题卡片。
3. 宽度小于 680px：一列主题卡片，快速操作堆叠，对话框接近全屏。
4. 高度不足时仅中部区域滚动，头部关闭按钮和底部常驻开关始终可达。

Codex Desktop 是主要目标，不为移动浏览器增加独立导航或触摸手势。

## 组件边界

保留 `buildSkinMenuScript` 作为唯一注入入口，在注入脚本内部建立以下边界：

1. `createThemeTrigger`：入口按钮、缩略图和隐藏状态。
2. `createThemeCenter`：遮罩、对话框、焦点管理和响应式容器。
3. `createCurrentThemeHero`：当前横幅与保存状态。
4. `createThemeCard`：内置、自定义和原生预览卡片。
5. `createQuickActions`：原生恢复与上传入口。
6. `createPersistenceFooter`：常驻、确认和隐藏入口。
7. `previewFromGeneratedCss`：严格提取内置主题 hero，失败时生成配色降级图。

这些函数只操作本次 generation 拥有的 DOM。reinjection 和 `dispose` 继续移除旧 generation 的监听器、计时器、网络请求和节点。

## 错误处理

1. 缩略图提取失败只降级预览，不影响主题应用。
2. 图片上传、解码、压缩和存储错误继续显示在自定义主题区域，不写入正式主题 state。
3. 主题提交失败恢复原主题、解除卡片忙碌状态并展示安全文案。
4. 组件创建中途失败不得删除当前有效皮肤；注入失败由现有 injector 结果报告。
5. 错误文案不得包含控制 token、完整本机路径、端点或环境变量。

## 可访问性

1. 主题中心使用 `role="dialog"` 和 `aria-modal="true"`。
2. 标题和说明通过 `aria-labelledby`、`aria-describedby` 关联。
3. 所有主题和操作使用原生 `button`。
4. 选中状态使用 `aria-pressed`，保存状态使用适度的 `aria-live="polite"`。
5. 删除自定义主题具有包含主题名的独立 `aria-label`。
6. 键盘可完成打开、浏览、切换、关闭、常驻确认和删除。
7. 不只依赖青色或樱粉区分选中状态，同时提供勾选图标和文字。

## 测试与验收

### DOM 单元测试

1. 入口包含当前主题缩略图、正确的展开状态和可恢复隐藏目标。
2. 对话框具备正确语义，打开后聚焦、焦点循环、`Esc`、遮罩和关闭按钮行为正确。
3. 三类预览来源分别覆盖内置 hero、自定义图片和原生降级卡。
4. hero 提取失败使用配色降级预览，不改变主题选择功能。
5. 卡片点击立即更新 renderer，面板不关闭，持久化在 ACK 前不发生。
6. 成功 ACK、冲突、HTTP 拒绝、renderer fallback 超时和更高 revision 均正确更新或回滚卡片、横幅和存储。
7. 自定义主题删除不会误触卡片选择。
8. 常驻确认、跨窗口广播、reinjection、dispose 和旧 generation 失效测试继续通过。

### CSS 与响应式测试

1. 大型对话框、固定头尾和滚动中区的关键规则存在。
2. 三列、两列和一列断点均存在。
3. 卡片图片区域、选中描边、勾选状态和降级预览样式存在。
4. 不引入外部 URL、网络字体、动画或新的持续计时器。

### 实机验收

1. 在真实 Codex 主 renderer 中打开主题中心，入口不遮挡窗口控制区。
2. 10 个内置主题都有可辨认缩略图。
3. 连续切换至少 6 次，视觉变化维持毫秒级，主题中心不关闭。
4. 后台 ACK 后 state revision、selectedThemeId、卡片、横幅和跨窗口状态一致。
5. 强制制造 revision 冲突时恢复原主题。
6. 上传一张图片后立即出现自定义卡片，重启后按现有本机存储契约恢复。
7. 检查助手透明背景、用户气泡和输入框玻璃样式没有回归。
8. 完整 `npm test` 为零失败，再重新构建确定性 `.skill` 包并安装到本机验证。

## 不在本轮范围

1. 主题搜索、分类、收藏、排序和远程主题市场。
2. 多张自定义图片历史。
3. 网络图片、云同步和跨设备同步。
4. 自动轮播主题、定时换肤或持续动画。
5. `diagnose --fix` 和其他进程、权限、锁状态自动修复能力。
