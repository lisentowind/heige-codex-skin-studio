# 全预设原生浅色表面兼容实施计划

**目标：** 用一个轻量公共 CSS 修复全部深色预设在 Codex 原生白色表面上的文字对比度，并完成 12 套主题真机回归。

**架构：** 保留各主题现有正文色，在公共皮肤样式中只为经过确认的原生浅色容器指定固定深色墨色。无新增运行时代码、依赖、监听器或模糊。

## Task 1：锁定回归合同

1. 在 `test/skin-css.test.mjs` 中要求输出原生浅色表面墨色变量。
2. 要求「环境信息」面板和顶部浅色按钮的窄范围选择器存在。
3. 要求规则不得覆盖面板内所有元素，保护增删统计等语义颜色。
4. 运行测试并确认先失败。

## Task 2：实现公共兼容层

1. 在 `src/skin-css.mjs` 根变量中加入固定墨色。
2. 加入两个窄范围选择器。
3. 运行皮肤 CSS 和相关回归测试，确认转绿。

## Task 3：逐套真机回归

1. 从发布候选包重新安装。
2. 依次应用 12 套内置主题，遇到控制器短暂锁时做有界重试。
3. 复核主题中心横幅、主题卡片、字体、看板、对话区和阅读增强。
4. 对深色主题读取原生白色表面的计算样式，确认文字为深色且语义色未被覆盖。

## Task 4：重建与发布

1. 更新变更记录和验证报告。
2. 运行全量测试、来源门禁和两次确定性打包。
3. 更新现有 PR，等待四类 GitHub 检查通过。
4. 合并 `main`，创建 `v5.4.0` Release，下载远端制品复核 SHA-256 和主题内容。
